Overview

who does what, at which scope

One org (company) per deployment in practice. Every role acts inside a fixed scope — the server re-derives it on every request. Drivers report to the Site Manager, not to a Team Head.

Chain of visibility & approval

Everything a lower role writes rolls up to the roles above it. Requests flow up; decisions flow down.

Site Manager

scope: exactly one site · invisible to every other site

The site's command center. Hard rule: an SM sees nothing — not a row, not a name — of any other site. (Enforced by per-request scope checks; the DB's row-security is org-level, so every SM read gets an isolation test.)
